# Security Policy

This repository currently contains a specification and process documents, not a production implementation.

## What To Report

Security-relevant reports are still useful when they concern:

- unsafe protocol guidance
- patterns that could lead reference implementations into insecure defaults
- accidental publication of secrets in the repository
- future example code once implementations are added

## Reporting Guidance

- Do not open a public issue for secrets or exploitable implementation details.
- Use a private channel to contact the maintainers when possible.
- If GitHub Security Advisories are enabled for the repository, prefer that path.

## Response Expectations

The project aims to:

- acknowledge reports quickly
- verify whether the issue affects the specification, examples, or workflow
- publish a fix or clarification in the spec when appropriate
